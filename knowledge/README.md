# Smart Procurement Navigator Knowledge Base Directory

This directory stores foundational materials required for the operation of the Smart Procurement Navigator. Currently a framework description is provided; actual files will be populated later by business or bid proposal leads.

## Required Knowledge Package

1. Enterprise qualification materials (Business license, professional Qualification certificate, safety production permit, etc.)
2. Personnel certificates and team overview
3. Project case library (including Contract, Acceptance report and client evaluation)
4. Historical response document library
5. Response document template library
6. Technical proposal material library
7. Business commitment and Contract clause materials
8. Common rejection risk checklist
9. Bid experience review Record
10. Technical specifications or requirements task statement text
11. Evaluation model and no-deviation index examples
12. Response document prohibited expression lexicon
13. Chart styles and historical chart materials
14. Government procurement regulations and local trading platform specification summaries

## Important Requirements

- Qualifications, performance records, and personnel information must be authentic and verifiable.
- Quotation and business commitments must be Confirmed by decision-makers.
- Demonstration examples use anonymized or fictitious demonstration projects; real client sensitive data must not be used.
- Reference response documents are for structural reference only; project names, locations, specific parameters, timelines, Quotation, and demonstration content must not be directly inherited.
- Internal evaluation strategy language, exaggerated rhetoric, and templated mechanical expressions must not remain in formal response texts.
- Comply with the relevant provisions of the "Government Procurement Law of the People's Republic of China" regarding information disclosure and fair competition.
